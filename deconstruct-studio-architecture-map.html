🏗️ 核心模块

ai-engine/ · AI 微服务

7 个 LLM 驱动的分析任务，串行调用链。每个任务接收/输出结构化 JSON。

deep_read → deconstruct → map_skeleton → check_similarity → strip_test → prompt_self_reflection → narrative_consistency_check

frontend/ · Web 前端

React + Tailwind 单页应用，实现 7 节点工作流完整交互。含 Playwright 端到端测试。

节点 1-7 状态机 · 冷却期锁死 · 灵气筛选 UI · 三问模态框 · 校验雷达图 · 头脑风暴白板

rule-engine/ · 规则引擎

冷却计时、频次控制、动态阈值放宽、三阶段状态切换。user_status API 作为所有操作的前置依赖。

cooldown_until · daily_submit_count · consecutive_failures · stage machine

guardian-ai/ · 守护服务

8 个自动化监控探针，24×7 运行。异常自动记录到 BigQuery。

驳回死循环 · 灵气污染 · 骨架单一 · 恶意刷 · 速率熔断 · 洗稿 · 离线降级 · 公共库污染

🔄 7 节点工作流（用户旅程）

节点1 选文导入 → 节点2 AI深度精读 → 节点3 分层拆解 → 节点4 灵气筛选 🧑 → 节点5 生成骨架 → 节点6 仿写重构 🧑 → 节点7 校验复盘

🧑 = 人工核心节点 — AI 只有建议权，无决策权

🔗 AI 引擎调用链（串行）

deep_read → deconstruct → 🧑 人工筛选（节点4） → map_skeleton → 🧑 仿写（节点6） → check_similarity

并行/独立任务：strip_test（节点4触发） · prompt_self_reflection（节点7失败触发） · narrative_consistency_check（节点7校验）

📊 数据模型

表用途关键字段
users用户账号 + 阶段状态stage, cooldown_until, consecutive_failures, daily_submit_count, reputation_score
deconstruct_sessions单篇拆解完整记录节点时间戳、AI输出路径、三问答案、校验历史
imitation_drafts仿写稿版本控制全文历史、相似度结果、防刷对比
skeleton_library公共骨架库topology_cluster_id, contributor_reputation
inspiration_entries个人灵气感知库笔记向量（Milvus）、手法抽象描述

🧭 三阶段成长体系

阶段训练配比拆解频次校验阈值
🌱 新手7仿写 3原创每 2-3 天一篇红灯 >70%
🌿 成长5仿写 5原创每周 ≤3 篇红灯 >60%
🌳 成熟100% 原创无限制仅记录

阶段切换条件：

新手→成长：
① 连续 5 篇仿写相似度均值 < 50%
② 情境剥离测试器使用率 < 20%

成长→成熟：
① 连续 30 天 AI 主动调用率 < 5%
② 提交纯原创代表作

👁️ Guardian-AI 探针矩阵

探针检测逻辑触发动作
驳回死循环2h 内 ≥3 次提交-驳回且无通过自动放宽阈值 10%
灵气库污染笔记-手法相似度 <0.5 比例 >30%推送健康检查通知
骨架拓扑单一编辑距离聚类 <4 种拓扑自动生成补全任务
恶意刷失败1h 内连续失败且稿间相似度 >80%冻结放宽资格
速率/成本熔断10min 内相似度请求 >10 次且全红/黄冻结 API 1h + CAPTCHA
洗稿检测仿写稿与网络内容子串匹配 >70%标记高风险，人工抽检
